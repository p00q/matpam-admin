\# grill-with-docs



목적:

작업 시작 전 요구사항, 영향 범위, 기준 파일, 대상 파일을 먼저 정렬한다.



절차:

1\. 바로 코드를 수정하지 않는다.

2\. 관련 Controller, Service, Mapper, JSP, JS, CSS 파일을 먼저 찾는다.

3\. 기존 구현 패턴과 메뉴 구조를 확인한다.

4\. 요구사항이 불명확하면 질문한다.

5\. 작업 계획을 작성한다.

6\. 필요하면 CONTEXT.md와 docs/adr/ 갱신안을 제안한다.

7\. 사용자가 승인하기 전에는 실제 파일을 수정하지 않는다.



출력:

\- 기준 파일

\- 대상 파일

\- 영향 범위

\- 확인 질문

\- 작업 계획
